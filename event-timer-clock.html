How it works: The ESP32 queries the DS3231 RTC once per second via I²C, computes elapsed time since the stored event timestamp, and renders it on the OLED. The RTC keeps time with ±2 ppm accuracy even when power is lost (coin cell backup).

Remote Automation Flow
💬 Slack Command or Google Form
Trigger Source A team member runs /reset-timer 2024-09-01 09:00 in Slack, or submits a Google Form with the new event date. Both feed into the automation platform.
webhook
⚙️ n8n / Make.com automation layer
Automation Platform n8n or Make.com receives the trigger, validates the date format, optionally logs to a Google Sheet, then fires an HTTP POST to the device's local API endpoint.
HTTP POST
📡 POST /api/reset {"ts": "2024-09-01T09:00:00"}
REST Endpoint on ESP32 The ESP32 exposes POST /api/reset over the local network. Body: {"ts":"ISO-8601"}. The firmware writes the timestamp to NVS flash and updates the DS3231 alarm register.
I²C write
⏱️ Device Resets new epoch saved to RTC
Timer Reset Complete The ESP32 stores the new event epoch in both NVS (survives firmware updates) and the DS3231's NVRAM (survives power loss). The OLED immediately begins counting from 0d 0h 0m 0s.
display
✅ OLED Updated 0d 0h 0m 0s → counting
Confirmation Within one second the display shows the new timer at zero and begins counting. The HTTP response returns 200 {"ok":true,"epoch":…} which n8n can use to post a Slack confirmation message.
